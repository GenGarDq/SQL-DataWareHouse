# Data WareHouse & Analytics Project

Добро пожаловать в мой проект по демонстрации лучших практик в построении хранлища данных и аналитики!
-- 

## Требования проекта
### Построить хранилище данных (Data WareHouse)

#### Цель
Разработать современное хранилище данных, используя PostgreSQL для консолидации данных о продажах, что позволит создать аналитические отчёты и принмать обоснованные решения. Как ELT инструмент используется dbt (data build tool), для визуализации - Apache Superset.

#### Спецификация

- **Источники данных**: Импорт данных из CRM (система управления взаимоотношений с клиентами) и ERP (система планирования ресурсов предприятия), предоставленных в виде CSV-файлов.
- **Качество данных**: очистка и устранение проблем с качеством данных перед анализом.
- **Интеграция**: объединение обоих источников в единую, удобную для пользователя модель данных, предназначенную для аналитических запросов.
- **Область применения**: Сосредоточение внимания только на самом последнем наборе данных; историзация данных не требуется.
- **Документация**: Предоставление четкой документации по модели данных для поддержки как заинтересованных сторон бизнеса, так и аналитических групп.
---

## Архитектура
Архитектура проекта следует "Медальонной" схеме и разделяет хранилище на **Bronze**, **Silver** и **Gold** слои данных.
1. Bronze слой содержит данные из CSV файлов источника без изменений.
2. Silver слой содержит очищенные, нормализованные, стандартизированные данные, готовые для дальнейшей аналитики.
3. Gold слой содержит витрины данных по схеме "Звезда" для целей бизнеса и аналитики.

## Запуск проекта

### Требуемое ПО
- Docker dekstop
- DBeaver (или любой другой инструмент взаимодействия с базой данных)
- VSCode (или любая другая IDE)

### Инициализация
1. Сохраните себе проект и перейдите с помощью консоли (Bash, PowerShell) в корень проекта (папка **SQL-DataWareHouse**)
    ```Bash
    cd SQL-DataWareHouse
    ```
2. Вам потребуется менеджер пакетов `pip` - его можно получить при установке python или же скачать отдельно. У меня уже был в комплекте с python. Далее выполните команду:
    ```Bash
    python.exe -m pip install --upgrade pip &&
    pytnon -m venv venv &&
    source venv/Scripts/activate
    ```
   Этим вы установите последнюю версию pip, создадите виртуальное окружение проекта (понадобится для dbt) и активируете его.

3. Выполните команду:
    ```Bash
    pip install requirements.txt
    ```
    Так вы подтяните все необходимые зависимости для dbt.

4. Запустите docker desktop и выполните команду:
    ```bash
    docker compose up -d
    ```
    В результате вы локально у себя развернёте базу данных PostgreSQL и Bi-инструмент Apache Superset.

4. Выполните команду:
    ```bash
    dbt init
    ```
    Консоль будет вас по очереди спрашивать параметры подключения к postgreSQL (можно посмотреть в docker-compose), они таковы:
    - host: localhost
    - port: 5432
    - user: postgres
    - password: postgres
    - dbname: DataWareHouse
    - schema: public (в процессе работы будем использовать другие)
    - threads: 2
    
    Затем перейдите в папку с dbt-проектом:
    ```bash
    cd dbt_dwh
    ```
    и выполните:
    ```bash
    dbt debug
    ```
    Должно вывести сообщение вида `All checks passed!`

**На этом приготовления завершены и проект готов к работе**
