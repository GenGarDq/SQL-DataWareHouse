# Соглашения об именованиях

Этот документ описывает соглашения об именованиях всех объектов в DWH: таблиц, представлений, процедур, схем и т.д.
--

## **Общие принципы**
1. Используется snake_case, все буквы строчные, слова разделяются знаком подчёркивания "_".
2. Использумый язык: английский для всех наименований.
3. Избегайте SQL-зарезервированных ключевых слов.

## Принципы именования таблиц
### Bronze-слой
- Все имена должны начинаться с имени источника и имени таблицы без каких-либо изменений. 
- Общий шаблон: **`<source>_<entity>`**
    - **`<source>`** - имя исходной системы (например: crm)
    - **`<entity>`** - имя таблицы из исходной системы
    - Пример: **`crm_customer_info`** - информация о покупателях из crm-системы.

### Silver-слой
- Все имена должны начинаться с имени источника и имени таблицы без каких-либо изменений. 
- Общий шаблон: **`<source>_<entity>`**
    - **`<source>`** - имя исходной системы (например: crm)
    - **`<entity>`** - имя таблицы из исходной системы
    - Пример: **`crm_customer_info`** - информация о покупателях из crm-системы.

### Gold-слой
- Все имена должны использовать осмысленное, связанное с бизнесом имя для таблицы, начинающееся с префикса категории.
- **`<category>_<entity>`**
  - `<category>`: описывает тип таблицы, например `dim` (таблица измерений) или `fact` (таблица фактов).  
  - `<entity>`: описывает имя, связанное с бизнес-сущностью (например, `customers`, `products`, `sales`).  
  - Примеры:
    - `dim_customers` - таблица измерений для данных покупателей.  
    - `fact_sales` - таблица фактов с данными продаж.

## Принципы наименований столбцов

### Суррогатные ключи 
- Все первичные ключи в таблицах измерений должны использовать суффикс `_key`.
- **`<table_name>_key`**
  - `<table_name>` - относится к имени таблицы или сущности, которой ключ принадлежит
  - `_key` -  суффикс, показывающий что эта колонка - суррогатный ключ.  
  - Пример -  `customer_key` - суррогатный ключ таблицы `dim_customers`.

### Технические столбцы
- Все технические столбцы должны начинаться с префикса `dwh_`, за которым следует описательное наименование столбца.
- **`dwh_<column_name>'**
  - `dwh` - префикс для системной метадаты.
  - `<column_name`> - описательное имя для столбца, показывающее его назначение.
  - Пример - `dwh_load_date` - системный столбец, используемый для хранения даты записи.

## Хранимые процедуры
- Все хранимые процедуры должны начинаться с префикса `load_`, за которым следует имя слоя, в который данные загружаются.
- **`load_<layer>`**.
  - `<layer>` - слой, куда загружаются данные (bronze, silver, gold).
  - Пример - `load_bronze` - хранимая процедура для загрузки данных в Bronze-слой.
