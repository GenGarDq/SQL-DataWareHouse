# Data WareHouse & Analytics Project

Добро пожаловать в мой проект по демонстрации лучших практик в построении хранилища данных и аналитики!
-- 

## Требования проекта
### Построить хранилище данных (Data WareHouse)

#### Цель
Разработать современное хранилище данных, используя PostgreSQL для консолидации данных о продажах, что позволит создать аналитические отчёты и принмать обоснованные решения. Как ELT инструмент используется dbt (data build tool).

#### Спецификация

- **Источники данных**: Импорт данных из CRM (система управления взаимоотношений с клиентами) и ERP (система планирования ресурсов предприятия), предоставленных в виде CSV-файлов.
- **Качество данных**: очистка и устранение проблем с качеством данных перед анализом.
- **Интеграция**: объединение обоих источников в единую, удобную для пользователя модель данных, предназначенную для аналитических запросов.
- **Область применения**: Сосредоточение внимания только на самом последнем наборе данных; историзация данных не требуется.
- **Документация**: Предоставление четкой документации по модели данных для поддержки как заинтересованных сторон бизнеса, так и аналитических групп.
---

## Архитектура
Архитектура проекта следует "Медальонной" схеме и разделяет хранилище на **Bronze**, **Silver** и **Gold** слои данных.
1. Bronze слой содержит данные из CSV файлов источника без изменений.
2. Silver слой содержит очищенные, нормализованные, стандартизированные данные, готовые для дальнейшей аналитики.
3. Gold слой содержит витрины данных по схеме "Звезда" для целей бизнеса и аналитики.

## Структура каталогов проекта
1. В папке `datasets` находятся .csv файлы из ERP и CRM - систем.
2. В папке `dbt_dwh` находятся файлы dbt-проекта.
   1. Папка `analyses` предназначена для ad-hoc запросов, которые не требуют материализации. (в данном проекте не используется)
   2. Папка `macros` содержит макросы (функции), которые позволяют переиспользовать фрагменты кода во избежание повторений.
   3. Папка `models` содержит sql-скрипты с jinja-шаблонами для заполнения слоёв хранилища, а также конфигурационные yaml-файлы
   4. Папка `seeds` предназначена для справочников небольшого размера (в данном проекте не используется)
   5. Папка `snapshots` предназначена для хранения версий моделей (историчность) (в данном проекте не используется)
3. Папка `Docs` содержит документацию по проекту
4. Папка `Scripts` содержит скрипт по созданию и наполнению bronze-слоя
5. В `docker-compose.yaml` содержится конфигурация создания БД PostgreSQL
6. `requirements.txt` содержит необходимые зависимости для проекта

## Запуск проекта

### Требуемое ПО
- Docker dekstop
- DBeaver (или любой другой инструмент взаимодействия с базой данных)
- VSCode (или любая другая IDE)
- Интерпретатор Python 3.12+

### Инициализация
1. Сохраните себе проект и перейдите с помощью консоли (Bash, PowerShell) в корень проекта (папка **SQL-DataWareHouse**)
    ```Bash
    cd SQL-DataWareHouse
    ```
2. Вам потребуется менеджер пакетов `pip` - его можно получить при установке python или же скачать отдельно. У меня уже был в комплекте с python. Далее выполните команду:
    ```Bash
    python.exe -m pip install --upgrade pip &&
    pytnon -m venv venv &&
    source venv/Scripts/activate
    ```
   Этим вы установите последнюю версию pip, создадите виртуальное окружение проекта (понадобится для dbt) и активируете его.

3. Выполните команду:
    ```Bash
    pip install requirements.txt
    ```
    Так вы подтяните все необходимые зависимости для dbt.

4. Запустите docker desktop и выполните команду:
    ```bash
    docker compose up -d
    ```
    В результате вы локально у себя развернёте базу данных PostgreSQL.

5. Выполните команду:
    ```bash
    dbt init
    ```
    Консоль будет вас по очереди спрашивать параметры подключения к postgreSQL (можно посмотреть в docker-compose), они таковы:
    - host: localhost
    - port: 5432
    - user: postgres
    - password: postgres
    - dbname: DataWareHouse
    - schema: public (в процессе работы будем использовать другие)
    - threads: 2
    
    Затем перейдите в папку с dbt-проектом:
    ```bash
    cd dbt_dwh
    ```
    и выполните:
    ```bash
    dbt debug
    ```
    Должно вывести сообщение вида `All checks passed!`

**На этом приготовления завершены и проект готов к работе**

## Заполнение таблиц
1. В консоли, находясь в корне проекта, выполните:
    ```Bash
    python ./Scripts/load_data_bronze.py
    ```
    Так вы создадите схемы для будущих слоёв и заполните bronze слой данных.
2. Затем выполните команду:
    ```Bash
    cd dbt_dwh/ && dbt build
    ```
    Так вы заполните silver и gold слои.

## Документация
Вы можете воспользоваться схемами и соглашениями в папке `Docs` в корне проекта.
Также для ознакомления с документацией по слоям, находясь в папке dbt_dwh, в консоли выполните:
    ```Bash
    dbt docs generate && dbt docs serve
    ```
Документация откроется автоматически в вашем браузере.
